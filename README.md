# 🎥 Azure Multi-Modal Compliance Orchestration Engine using LangGraph and Langsmith

## 📌 Overview

This project establishes an automated Video Compliance QA Pipeline orchestrated by LangGraph, designed to audit content against regulatory standards using a RAG architecture. Leveraged Azure Video Indexer for multimodal ingestion (transcripts/OCR) and Azure AI Search to retrieve relevant compliance rules via Azure OpenAI Embeddings. The core reasoning engine, Azure OpenAI (GPT-5), synthesizes this data to deterministically detect violations, while LangSmith provides granular tracing for LLM workflow optimization.
Additionally, Azure Application Insights is integrated for production-grade telemetry, logging, and real-time performance monitoring. This end-to-end system transforms unstructured video into structured, actionable JSON compliance reports with deep full-stack observability..

## 🚀 Key Features

* Automated compliance auditing for video content.
* Multimodal ingestion using Azure Video Indexer.

## ⚙️ Workflow

### 1. Video Ingestion

The uploaded video is processed through Azure Video Indexer to extract:

* Speech transcripts
* OCR text
* Temporal metadata

### 2. Compliance Retrieval

Relevant compliance policies are retrieved from Azure AI Search using semantic embeddings generated by Azure OpenAI.

### 3. Intelligent Reasoning

LangGraph orchestrates the reasoning workflow, where GPT-4o analyzes:

* Video transcript
* OCR content
* Retrieved compliance rules

### 4. Violation Detection

The system identifies:

* Compliance violations
* Rule references
* Severity levels
* Explanations

### 5. Report Generation

A structured JSON report is generated for downstream systems.

Example:

```json
{
  "video_id": "vid_001",
  "status": "non_compliant",
  "violations": [
    {
      "rule_id": "CMP-101",
      "severity": "high",
      "timestamp": "00:03:42",
      "description": "Unauthorized claim detected."
    }
  ]
}
```

### 6. Observability

The entire workflow is monitored using:

* LangSmith traces
* Token usage analytics
* Latency monitoring
* Error tracking
* Azure Application Insights dashboards
